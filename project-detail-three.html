Petcare Service Website

UI/UX Design

Date

April 13th, 2022

Role

Graphic Designer

TOOLS

Adobe Illustration, Adobe Indesign, Figma, Maze

Members

Individual

Project Overview

The main goal of this project is to use Figma to construct an interactive prototype (a desktop website) for a fictitious firm. Searching, browsing, bookmarking, purchasing, checkout, login, user setup, message, and so on should all be included in the prototype.

01 Ideatation & Information
Gathering

Information Gathering

The project's details and content are entirely fictitious. Images are royalty-free and were obtained from Unsplash and Adobe Stocks. During the implementation process, ensure that the visuals and information in the project are appropriate and compatible with the project's content.

User Scenario

I design user scenarios to better understand people so that I can build more effective user flows. User Scenarios assist me in mapping out user pain points, motivations, and product development potential.

02 Work Process

User Flow

The most difficult stage is creating the user flow. The design must ensure that when users use the website, they feel straightforward and easy to comprehend, and it must restrict the complexity when constructing an actual website.

Wireframes & Prototype

Low Fidelity Wireframes

Low-fidelity wireframes are used to generate ideas for hi-fi wireframes.

PetVet is a pet-care website. The website contains basic services like pet health monitoring, pet appointment booking, and pet spa, which make it easier for consumers to care for their pets.

High Fidelity Wireframes

High-fidelity wireframes are generate from low-fi wireframes.

To guarantee that the interactive layout works properly, prototypes are tested. All of the buttons and links work properly and lead to both internal and external pages.

User Testing

This is a critical stage of the project. A list of tasks will be assigned to the testers, which they must perform. Designers can gain an overview of the design's flaws based on the miss-click rate and task time. Potential users' feedback will be collected (through Maze), allowing me to make required changes to the final product.

03 Final Product

The final product expected to look like a real finished product (including content and visuals). 14 screens were designed to demonstrate the website flow and how the user interaction on the website. Logo was created using Adobe Illustration; and all the hi-fi wireframes with the prototype were designed using Figma; users test and report using Maze.
